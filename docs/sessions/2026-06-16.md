## 3. 🔴 EDINET統合バグ（最重要発見）

**「ここまでの成果」の中核であるEDINET統合が、本番・ローカルとも一度も機能していなかった。**

- **症状**: 日本株検索（7203トヨタ）で `edinetCode: null`、財務データが一切付かない（本番・ローカル両方で再現）。
- **切り分け**:
  - 発火条件（`route.ts` line 566-570）正常: `EDINETDB_API_KEY` あり ＋ `symbol.endsWith(".T")`
  - secCode抽出（line 574）正常: `"7203.T"` → `"7203"`
  - 実API実測: `GET /v1/companies?q=7203` → **HTTP 200・0.26秒・`data` に3件（トヨタ自動車含む）**
- **根本原因**: APIレスポンスは `{ data: [...], meta: {...} }` 形式だが、`edinetdb.ts` の全メソッドが `results` / `companies` キーを読んでいる:

  | 行 | メソッド | 現在のパース |
  |---|---|---|
  | 18-21 | 型 `EdinetSearchResponse` | `results?` / `companies?` のみ（`data` 無し） |
  | 38 | `searchCompanies` | `data.results ?? data.companies ?? []` |
  | 49 | `getCompany` | `detail.results?.[0] ?? detail.companies?.[0] ?? null` |
  | 55 | `getFinancials` | `data.results ?? data.companies ?? []` |
  | 60-66 | `getRatios` | 同パターン |

  → 全メソッドが常に空/null を返す → edinetCode も ratios も financialHistory も付かない。

- **なぜ気づかなかったか**:
  1. `Promise.race(...).catch(() => [])`（route.ts line 580）＋ `length > 0` スキップでエラー完全握りつぶし（ログすら出ない）
  2. 日本株で edinetCode が付くことの検証テストが無い
  3. EDINET無しでも検索はFMP/Yahooで成立しUIが壊れない → 「効いている」と誤認

- **その他**: `route.ts` line 82 は `SERPAPI_API_KEY` を読むが `.env` は `SERPAPI_KEY`（同種の env名不一致、要確認）。

### 修正方針（当時・原因確定済み）
1. `edinetdb.ts` のパースを `data` キー対応に統一（型に `data?` 追加、各 return を `data` 優先に）。getRatios/getCompany は配列/単一を実APIで確認
2. サイレント握りつぶしを改め、EDINET失敗時はログを残す
3. 回帰防止に最小テスト（7203 → edinetCode が付く）
